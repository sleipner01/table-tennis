Changes made


General changes

  • Color palette
    • The color palette was not chosen with dark mode in mind, so we had to introcude several more shades of the colors we had chosen to make it work well, but this didn't prove to be an issue.
    • The black color we had chosen ( HEX: #010101 ) was a bit too dark at times, especially for dark mode, so we opted for a black that was easier on the eyes ( HEX: #111111 ).
  • Favicon
    • We also made a "smaller" version of the logo, to serve as a favicon. This smaller version is far better suited to being an icon, because it is far simpler. This leads to it being recognizeable, even in the small form factor.
    • Besides it reuses elements from the original logo, which ties the whole theme together, and people can easily gather that it's the same organization using both.
  • Footer
    • We also added a dark mode button to the footer, which forced us to change the design a tad to make the flow of the footer work as we wanted, but we're really happy with the result.
  • Titles
    • We added titles to the top of each page, to ensure clarity for the user.

Specific file changes

  • index.html
    • We added a title and a button to “regular trainings” to improve user experience and interaction, with a central "Call to action" button the user can click.
    • We opted for only showing a week at a time on the index page as well, to not take up all the space, but still show a shorthand version of upcoming events.
    • On the bottom of the page we added some text with rackets to make it match the subject of the website.
  • intro-to-table-tennis
    • We gave each section it's own title so that the page is easier to scan for what content the user wants to read.
    • We changed the background color to red for the "rules" section, to highlight it. We felt the pop of color really fits with a such otherwise mundane page, and wanted to highlight the arguably most important section of the page.
  • regular-training
    • Added a row displaying weekdays.
    • Also made the current displayed month and year bigger, to avoid getting lost in the calendar.
  • social-gatherings
    • Replaced the clickable arrows to control the displayed year, to angle brackets to keep the site consitent.
    • Changed color of callapsible to red (Hex: #A8000F) instead of gray, to make the page look more alive.
    • Changed color of hover and active states to dark red (Hex: #8A000E), to give a slight contast to the new red color.
    • Changed layout of tournament content to only show participants and the score, instead of spesifcally telling who won. This made the content look cleaner and still informative.
    • Added header to event content. Gives the layout a natural look, and good readability.
    • Instead of "+" and "-" signs in the collapsibles, we went for keyboard-arrow- up/down signs from Material Icons. This was done to keep the site consistent.
  • contact
    • Of course we added placeholder text to show the user examples of input values, to make the process easier for them.
    • As well as the inputs being green once the values inside are valid, this helps the user find potential errors while filling the form.
  • shared_html.js
    • This one is completely new, and we added it after getting information that it is mandatory.
  • scoreboard.js
    • No changes to intended design.
  • training_table.js
    • No changes to intended design.
  • form_validation.js
    • If a user tries to submit an invalid form, this script will stop it, and tell the user where the error lies, with a short message above the input. This provides assistance to the user filling the form, and makes the process easier to understand.
  • hamburger_navbar.js
    • On page load the whole nav bar flashes white, and slowly fades. This is to make the navigation bar more noticeable for the user, but the background color fades to give more space on the page for content.
  • homepage_gallery.js
    • No changes to intended design.
  • gatherings_carousel.js
    • Added bars on the side of the page, to show the previous and next year's content. This way the user can see if the other years have content availible, without even having to "scroll" to them.
  • dark_mode.js
    • No changes to intended funcitonality.
  • Changes to intended CSS styling
    • We opted for using even more red on the pages, than we initially thought we would. This was to lead the viewers eyes comfortably, and outline which parts of the website we thought should have focus. And a lot of refining was done to make the whole website feel smoother, and more professional.
